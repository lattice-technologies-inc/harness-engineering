Read the shaping skill, then read the full breadboard reflection reference at the skill's `references/breadboard-reflection.md`.

You MUST read the entire reference file before proceeding.

After reading, analyze the breadboard:
1. Trace each user story through the wiring — does the path tell a coherent story?
2. Check for the smells listed in the reference (incoherent wiring, missing paths, naming resistance, etc.)
3. Apply the Naming Test to each affordance
4. Report findings and propose fixes
